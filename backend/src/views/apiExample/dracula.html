Criar conta de usuário
POST/auth/signup

Cria a conta de um usuário, podendo ser um contratante ou prestador.

Limite de acesso: Máxima de 5 tentativas, bloqueio de 120 segundos.

Campos obrigatórios - credentials ¶

  • name (string, mínimo 2 caracteres)

  • email (string, e-mail válido)

  • password (string, mínimo 8, máximo 125 caracteres)

  • phone (string, 10 a 15 caracteres)

Campos opcionais ¶

necessários adicionar como verdadeiro

  • accept_privacy_Policy (boolean) default: false
  • accept_terms_use (boolean) default: false

Campos opcionais: ¶

  • whatsapp (string, 10 a 15 caracteres)

  • allow_see_address (boolean) default: false

  • address (object): Obrigatório se o objeto provider for informado

    • cep (string, 8 caracteres)
    • city (string, 2 a 64 caracteres)
    • state (string, 2 a 64 caracteres)
    • street (string, 2 a 64 caracteres)
    • number (string, 1 a 8 caracteres)
    • latitude (string, 12 caracteres, opcional)
    • longitude (string, 12 caracteres, opcional)

A senha devem obedecer aos seguintes requisitos: ¶

  • Mínimo 8 caracteres.

  • Pelo menos uma letra minúscula.

  • Pelo menos uma letra MAIÚSCULA.

  • Pelo menos um número (0 a 9).

  • Pelo menos um caractere especial: !@#$%¨:;^&*()[]{}/*-+|\\=<>.

  • Não deve conter palavras-chave que facilitem adivinhação (ex.: “senha”, “1234”, “password”, “admin”).

Script para gerenciamento de senha ¶

// 250208V
// powerd by: LucasATS

const wordlistPass = [
    '123456', '123', 'teste123*',
    'senha', 'password', 'qwerty',
    'abc123', 'admin', 'iloveyou',
    'letmein', 'monkey', 'sunshine',
    'superman', 'welcome', 'senha'];
const caractereEspeciais = '!@#$%¨:;^&*()[]{}/*-+|\\=<>'.split('');
function validadePassword(senha) {
    if (senha.length < 8) return { success: false, error: 'A senha deve conter pelo menos 8 caracteres.', status: 422 };

    const letrasMinusculas = /[a-zà-ÿ]/.test(senha);
    if (!letrasMinusculas) return { success: false, error: 'A senha deve conter pelo menos uma letra minúscula.', status: 422 };

    const letrasMaiusculas = /[A-ZÀ-Ÿ]/.test(senha);
    if (!letrasMaiusculas) return { success: false, error: 'A senha deve conter pelo menos uma letra MAIÚSCULA.', status: 422 };

    const existeNumero = /\d/.test(senha);
    if (!existeNumero) return { success: false, error: 'A senha deve conter pelo menos um número de 0 a 9.', status: 422 };

    const existeCaracteresEspeciais = caractereEspeciais.some(caracter => RegExp(`\\${caracter}`).test(senha));
    if (!existeCaracteresEspeciais) return { success: false, error: 'A senha deve conter pelo menos um caractere especial !@#$%¨:;^&*()[]{}/*-+|\\=<>', status: 422 };

    const existePlavrasChaves = wordlistPass.some(palavra => RegExp(`\\${palavra}`).test(senha));
    if (existePlavrasChaves) return { success: false, error: 'A senha é facilmente adivinhável.', status: 422 };

    return { success: true, message: 'A senha é segura.', status: 200 };
}

Enviar OTP
POST/auth/send-otp

Gera e envia através de uma modal o código de segurança temporário (OTP).

Essa API pode ser usada para ATIVAR um E-mail ou ALTERAR uma senha, usando o OTP.

Limite de acesso: Máxima de 5 tentativas, bloqueio de 120 segundos.

O OTP é válido por 15 minutos e só pode ser reenviado após 120 segundos.

Protegido pelo middleware isAuth.

(OPCIONAL) O cabeçalho authorization poderá ser enviado no formato Bearer <token>, e o token autenticado é utilizado para ATIVAR o E-mail.

Por hora, só existe a possibilidade de enviar o código de segurança via e-mail.

Exemplo usando fetch ¶

const URL = 'https://dev-diboaclub.onrender.com';
const TOKEN = ''; // Deixe vazio para ALTERAR senha
const EMAIL = 'teste1@email.com'; // Defina um email válido se não houver token

if (!TOKEN && !EMAIL) {
    console.error("Token ou email são obrigatórios.");
} else {
    fetch(URL + '/auth/send-otp', {
        method: 'POST',
        headers: {
            "Content-Type": "application/json",
            ... (TOKEN ? { authorization: `Bearer ${TOKEN}` } : {})
        },
        body: JSON.stringify(TOKEN ? {} : { email: EMAIL }), // Envia o email apenas se não houver token
    })
    .then(resp => resp.json())
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error(error);
    });
}

Validar OTP/Alterar Senha
POST/auth/check-otp

Valida se o código OTP está correto. Se for para ativar o e-mail, ele será validado. Se for para alteração de senha, a senha será atualizada e um token de acesso (login) será gerado.

Limite de acesso: Máxima de 5 tentativas, bloqueio de 120 segundos.

Protegido pelo middleware isAuth.

(OPCIONAL) O cabeçalho authorization poderá ser enviado no formato Bearer <token>, e o token autenticado é utilizado para ATIVAR o E-mail.

O retorno 200 referece ao ato de ATIVAR email e o 201 ao ato de ALTERAR senha.

Exemplo usando fetch ¶

const URL = 'https://dev-diboaclub.onrender.com';
const TOKEN = ''; // Insira um token válido para testar com autenticação
const EMAIL = 'teste1@email.com'; // Insira um email válido para testar sem autenticação
const OTP = '<<CODE>>'; // Código OTP a ser verificado
const password = "<<PASSWORD>>";

if (!TOKEN && !EMAIL) {
    console.error("Token ou email são obrigatórios.");
} else {
    const bodyData = { otp: OTP };
    if (!TOKEN) bodyData.email = EMAIL; // Adiciona o email apenas se não houver token
    if (!TOKEN) bodyData.password = password; // Adiciona a senha se não houver token
    
    fetch(URL + '/auth/check-otp', {
        method: 'POST',
        headers: { 
            "Content-Type": "application/json",
            ...(TOKEN ? { authorization: `Bearer ${TOKEN}` } : {})
        },
        body: JSON.stringify(bodyData),
    })
    .then(resp => resp.json())
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error(error);
    });
}

Alterar foto de perfil
PUT/api/account/picture

Pode ser usado para enviar ou trocar a foto de perfil do usuário. As imagens são automaticamente recortadas para 512x512 e convertidas para WebP.

Limite de acesso: Máxima de 10 tentativas, bloqueio de 120 segundos.

Protegido pelo middleware isAuth.

Limite máximo de upload: 100mb. Tamanho máximo do arquivo: 4mb.

Não utilize o Content-Type: application/json no header dessa API. Não há necessidade de uso de Content-Type!

Utilize o método FormData para o envio de arquivos.

Exemplo usando fetch com formData ¶

const uploadProfilePicture = async () => {
  const URL = "https://dev-diboaclub.onrender.com";
  const TOKEN = "JWT.EXAMPLE.TOKEN";

  // Baixar uma imagem da web e transformá-la em um blob
  const imageUrl = "https://picsum.photos/1024/1024.jpg"; // API imagem aleatória
  const response = await fetch(imageUrl);
  const blob = await response.blob();
  const file = new File([blob], "picture.jpg", { type: "image/jpeg" });

  // Criar FormData
  const formData = new FormData();
  formData.append("picture", file);

  // Fazer a requisição para a API
  fetch(URL + "/api/account/picture", {
    method: "PUT",
    headers: {
      "authorization": `Bearer ${TOKEN}`,
    },
    body: formData,
  })
  .then(response => response.json())
  .then(data => console.log("Resposta:", data))
  .catch(error => console.error("Erro:", error));
};

// Executar upload
uploadProfilePicture();

const BASE_URL = "https://dev-diboaclub.onrender.com";
const API_URL = "/api/provider/file";
const TOKEN = "JWT.EXAMPLE.TOKEN"; // Substitua pelo token válido

async function request(method, fileUrl = null, fileName = null, fileId = null) {
    const options = {
        method,
        headers: { "authorization": `Bearer ${TOKEN}` },
    };

    if (fileUrl) {
        const formData = new FormData();
        const response = await fetch(fileUrl);
        const blob = await response.blob();
        formData.append("file", blob, fileName);
        options.body = formData;
    }

    if (fileId) options.method = "DELETE";

    const response = await fetch(BASE_URL + API_URL + (fileId ? `/${fileId}` : ""), options);
    const data = await response.json();
    console.log(`✅ ${method} ->`, data);
}
